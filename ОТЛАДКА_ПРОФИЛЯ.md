# 🔍 Отладка: Профиль не обновляется

## Шаг 1: Проверка базы данных

Запустите скрипт проверки:

```bash
cd backend
python check_profile_update.py
```

Скрипт проверит:
- ✅ Наличие всех необходимых колонок в БД
- ✅ Состояние профилей пользователей
- ✅ Папку для загрузки аватаров
- ✅ Права доступа

## Шаг 2: Проверка backend сервера

### 2.1 Убедитесь, что backend запущен

```bash
curl http://localhost:5001/api/health
```

Должен вернуть:
```json
{"status": "ok", "message": "API работает корректно"}
```

### 2.2 Проверьте endpoint профиля

Замените `1` на ID вашего пользователя:

```bash
# Получить профиль
curl http://localhost:5001/api/users/1/profile

# Обновить профиль
curl -X PATCH http://localhost:5001/api/users/1/profile \
     -F 'name=Тестовое Имя' \
     -F 'bio=Тестовое описание' \
     -F 'location=Минск'
```

## Шаг 3: Проверка в браузере

### 3.1 Откройте консоль разработчика

Нажмите `F12` или `Ctrl+Shift+I` (Cmd+Option+I на Mac)

### 3.2 Перейдите на вкладку Console

### 3.3 Попробуйте обновить профиль

Вы должны увидеть логи:
```
Отправка данных профиля: {name: "...", bio: "...", ...}
Ответ сервера: {status: "ok", ...}
Загрузка профиля для пользователя: 1
Данные профиля с сервера: {name: "...", ...}
```

### 3.4 Проверьте вкладку Network

1. Перейдите на вкладку `Network`
2. Попробуйте обновить профиль
3. Найдите запрос `PATCH /api/users/.../profile`
4. Проверьте:
   - Status Code (должен быть 200)
   - Response (ответ сервера)
   - Request Payload (отправленные данные)

## Шаг 4: Проверка логов backend

В терминале где запущен backend вы должны видеть запросы:

```
127.0.0.1 - - [06/Mar/2024 10:30:45] "GET /api/users/1/profile HTTP/1.1" 200 -
127.0.0.1 - - [06/Mar/2024 10:30:50] "PATCH /api/users/1/profile HTTP/1.1" 200 -
```

Если видите ошибки (500, 404, 400), проверьте текст ошибки.

## Шаг 5: Типичные проблемы и решения

### Проблема 1: "Column 'avatar' not found"

**Решение:**
```bash
cd backend
python migrate_db.py
python app.py
```

### Проблема 2: "Permission denied" при загрузке аватара

**Решение:**
```bash
chmod 755 backend/uploads/avatars
```

### Проблема 3: Backend не запущен

**Решение:**
```bash
cd backend
python app.py
```

Должно появиться:
```
 * Running on http://0.0.0.0:5001
```

### Проблема 4: Порт 5001 занят

**Решение:**
```bash
# Найти процесс
lsof -i :5001

# Или запустить на другом порту
PORT=5002 python app.py
```

Не забудьте изменить порт в `src/pages/Profile.js`:
```javascript
const API = 'http://localhost:5002';
```

### Проблема 5: CORS ошибка

В консоли браузера видите:
```
Access to fetch at 'http://localhost:5001/...' from origin 'http://localhost:3000' has been blocked by CORS policy
```

**Решение:**
Убедитесь, что в `backend/app.py` есть:
```python
CORS(app, resources={r"/api/*": {"origins": "http://localhost:3000"}})
```

### Проблема 6: Данные не сохраняются

**Проверьте:**
1. База данных не заблокирована
2. Есть права на запись в `backend/ecology.db`
3. Диск не заполнен

**Решение:**
```bash
# Проверить права
ls -la backend/ecology.db

# Исправить права
chmod 644 backend/ecology.db
```

### Проблема 7: Аватар не отображается

**Проверьте:**
1. Файл загружен в `backend/uploads/avatars/`
2. Backend отдает файлы по URL `/uploads/avatars/<filename>`

**Тест:**
```bash
# Проверить, что файл существует
ls -la backend/uploads/avatars/

# Попробовать скачать аватар
curl http://localhost:5001/uploads/avatars/1_1234567890_avatar.jpg
```

## Шаг 8: Тестовое обновление через curl

Попробуйте обновить профиль напрямую через curl:

```bash
# Замените 1 на ваш user_id
curl -X PATCH http://localhost:5001/api/users/1/profile \
     -F 'name=Новое Имя' \
     -F 'bio=Новое описание' \
     -F 'location=Новое место'

# Проверьте результат
curl http://localhost:5001/api/users/1/profile
```

Если через curl работает, а через браузер нет - проблема в frontend.
