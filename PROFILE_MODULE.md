# Модуль профилей пользователей

Модуль профилей пользователей представляет собой систему управления персональными данными и аватарами, обеспечивающую пользователям возможность настройки своего публичного представления на платформе.

## Архитектура

### Frontend компоненты

**Profile.js** (`src/pages/Profile.js`)
- Страница редактирования профиля пользователя
- Загрузка и предпросмотр аватара
- Редактирование имени, местоположения и биографии
- Валидация размера файла (макс. 5 МБ) и типа (JPG, PNG, GIF)
- Отображение инициалов при отсутствии аватара
- Поддержка двуязычности (RU/EN) и темной темы

**AvatarDisplay.js** (`src/components/AvatarDisplay.js`)
- Универсальный компонент отображения аватара
- Поддержка размеров: small (32px), medium (48px), large (80px), xlarge (120px)
- Автоматическая генерация инициалов из имени пользователя
- Градиентный фон для placeholder

### Backend API

**Endpoints:**

`GET /api/users/<user_id>/profile`
- Получение данных профиля пользователя
- Возвращает: id, name, email, bio, location, avatar, created_at

`PATCH /api/users/<user_id>/profile`
- Обновление профиля с загрузкой аватара
- Принимает multipart/form-data с полями: name, bio, location, avatar (файл)
- Автоматическое удаление старого аватара при загрузке нового
- Генерация уникального имени файла с timestamp

`GET /uploads/avatars/<filename>`
- Отдача файлов аватаров
- Статический endpoint для доступа к загруженным изображениям

### База данных

**Таблица users** (расширенная):
- `avatar` TEXT - путь к файлу аватара (/uploads/avatars/filename)
- `bio` TEXT - биография пользователя (макс. 500 символов)
- `location` TEXT - местоположение пользователя
- `created_at` TEXT - дата регистрации (ISO 8601)

### Файловая система

**Структура хранения:**
```
backend/
  uploads/
    avatars/
      {user_id}_{timestamp}_{filename}
```

## Интеграция

**Account.js** - добавлена кнопка "Редактировать профиль" и отображение аватара в шапке аккаунта

**App.js** - добавлен маршрут `/profile` для страницы профиля

## Безопасность

- Валидация типов файлов (только изображения)
- Ограничение размера файла (5 МБ)
- Использование `secure_filename` для безопасного сохранения
- Уникальные имена файлов с user_id и timestamp
- Автоматическое удаление старых аватаров

## Особенности

- Email нельзя изменить (отображается как disabled поле)
- Автоматическая генерация инициалов из имени (первые буквы имени и фамилии)
- Предпросмотр аватара перед загрузкой
- Счетчик символов для биографии (макс. 500)
- Адаптивный дизайн для мобильных устройств
- Поддержка темной темы с градиентными фонами
